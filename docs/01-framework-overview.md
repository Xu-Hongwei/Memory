# 01. 记忆框架总览

## 1. 核心定义

智能体记忆系统可以拆成五类对象。

### 1.1 Event

`Event` 是发生过的原始事件，例如：

- 一轮用户消息
- 一次工具调用结果
- 一段文件读取结果
- 一次测试失败
- 一次用户确认

Event 不等于长期记忆。它只是原始证据。

### 1.2 Observation

`Observation` 是从事件中提取出的观察，例如：

- 用户希望默认使用中文
- 某项目的启动命令是 `npm run dev`
- 某接口在端口 `3000` 上响应
- 某个错误由环境变量缺失导致

Observation 仍然可能是临时的、片面的或未经验证的。

### 1.3 Memory Candidate

`Memory Candidate` 是候选记忆。它需要经过写入门禁判断。

候选记忆应该回答：

- 它是什么类型
- 适用于什么范围
- 是否长期有用
- 是否有证据
- 是否可能敏感
- 是否和已有记忆重复或冲突

### 1.4 Memory Item

`Memory Item` 是已经被确认写入的长期记忆。它必须有结构化元数据。

典型字段包括：

- 内容
- 类型
- 范围
- 来源
- 置信度
- 最近验证时间
- 生效状态
- 版本信息

### 1.5 Memory View

`Memory View` 是为当前任务动态组装的记忆上下文。

它不是全部记忆，而是根据当前任务检索、压缩、排序后的最小必要上下文。

## 2. 记忆分层

### 2.1 Working Memory

当前任务上下文，只在当前会话或当前任务内有效。

适合保存：

- 当前用户目标
- 当前代码修改计划
- 当前临时假设
- 当前未完成步骤

不适合写入长期记忆。

### 2.2 Episodic Memory

事件型记忆，记录“发生过什么”。

适合保存：

- 某次排错过程
- 某次用户确认
- 某次项目迁移
- 某次测试结果

Episodic Memory 适合作为证据来源，但不一定直接注入每次对话。

### 2.3 Semantic Memory

语义型记忆，保存稳定事实。

适合保存：

- 用户长期偏好
- 项目架构事实
- 模块职责
- 环境约束
- 业务规则

这是最常用的长期记忆层。

### 2.4 Procedural Memory

流程型记忆，保存“怎么做”。

适合保存：

- 工具调用规则
- 项目启动流程
- 发布流程
- 排错步骤
- 代码审查准则

这类记忆对智能体执行任务非常重要。

### 2.5 Reflective Memory

反思型记忆，保存从多次经验中总结出来的高层经验。

适合保存：

- 某类问题的共性原因
- 某个用户稳定的协作方式
- 某个项目长期容易出错的点
- 多次排错后的最佳路径

Reflective Memory 不应该由单次事件直接生成，最好由定期巩固任务生成。

## 3. 推荐记忆类型

| 类型 | 说明 | 示例 |
| --- | --- | --- |
| `user_preference` | 用户长期偏好 | 默认中文、喜欢先验证再结论 |
| `project_fact` | 项目事实 | 后端入口、配置路径、模块职责 |
| `tool_rule` | 工具规则 | 某工具需要先搜索再写入 |
| `environment_fact` | 环境约束 | Windows PowerShell、特定路径、端口约定 |
| `troubleshooting` | 已验证排错经验 | 问题、经验、解决方式 |
| `decision` | 历史决策 | 为什么采用某架构或命名 |
| `workflow` | 可复用流程 | 构建、测试、发布、回滚 |

## 4. 记忆生命周期

一条信息从出现到成为长期记忆，建议经历以下生命周期：

```text
Raw Event
  -> Observation
  -> Candidate Memory
  -> Policy Check
  -> Verification
  -> Commit
  -> Retrieval
  -> Reuse
  -> Revalidate
  -> Update / Archive / Forget
```

## 5. 基本边界

### 5.1 不应该写入的内容

- 一次性请求
- 当前临时状态
- 未验证猜测
- 普通寒暄
- 敏感信息
- 可从当前上下文直接获得的信息
- 已经存在且无增量的信息

### 5.2 应该优先写入的内容

- 用户明确表达的长期偏好
- 项目中稳定且经验证的事实
- 工具和环境的可复用规则
- 已真实发生并已解决的排错经验
- 未来很可能再次影响决策的信息
